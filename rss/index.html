<code>기본 설정</code>을 클릭하여 <code>앱 ID</code>와 <code>앱 시크릿 코드</code>를 확인한다.</p> <p><img src="/images/fb_03_01_05.png" alt="설정 - 기본 설정" width="30%" /></p> <p><img src="/images/fb_03_01_06.png" alt="앱 ID와 앱 시크릿 코드 확인" width="30%" /></p> </li> <li> <p>하단의 <code>+ 플랫폼 추가</code> 버튼을 클릭하고 iOS나 Android 등의 플랫폼을 선택한다(여기서는 iOS를 기준으로 설명한다). <code>번들 ID</code>를 입력한다. 다 입력하면 하단의 <code>변경 내용 저장</code>을 클릭한다.</p> <p><img src="/images/fb_03_01_07.png" alt="iOS 번들 ID 입력" width="80%" /></p> </li> <li> <p>이제 아대시보드로 접속하여 <code>로그인 방법</code>탭을 선택한 뒤 <code>Facebook</code>을 클릭한다.</p> </li> <li> <p><code>사용 설정</code>을 클릭하고 앞서 확인한 <code>앱 ID</code>와 <code>앱 시크릿 코드</code>를 입력한다. <code>OAuth 리디렉션 URI</code>를 복사한다(복사 아이콘 클릭)하고 <code>저장</code> 버튼을 클릭한다.</p> <p><img src="/images/fb_03_01_08.png" alt="앱 ID, 앱 시크릿 코드 입력" width="60%" /></p> </li> <li> <p>facebook for developers 대시보드로 돌아가서 복사한 <code>OAuth 리디렉션 URI</code>를 <code>Facebook 로그인</code> - <code>설정</code>에서 <code>유효한 OAuth 리디렉션 URI</code>에 붙여넣고 <code>변경 내용 저장</code>을 클릭한다.</p> <p><img src="/images/fb_03_01_09.png" alt="좌측 메뉴의 Facebook 로그인 - 설정" width="30%" /></p> <p><img src="/images/fb_03_01_10.png" alt="유효한 OAuth 리디렉션 URI 입력" width="40%" /></p> </li> </ul> <h2 id="facebook-로그인">Facebook 로그인</h2> <p>Facebook 로그인도 익명 로그인과 비슷한 절차로 진행된다.</p> <ul> <li>사용자) Facebook 로그인 버튼 클릭</li> <li>게임 클라이언트) Facebook SDK를 활용해서 로그인 화면 표시</li> <li>사용자) 로그인 진행</li> <li>Facebook) access token 반환</li> <li>게임 클라이언트) access token을 Firebase 인증 SDK를 통해 등록</li> <li>Firebase) 사용자 등록 과정 진행</li> <li>게임 클라이언트) 고유한 사용자 ID를 사용</li> </ul> <h3 id="프로그래밍-준비">프로그래밍 준비</h3> <blockquote> <p><a href="http://totuworld.github.io/2018/03/22/firebaseandunity-01/">익명 로그인</a>을 진행했다면 그 프로젝트를 이어서 사용해도 무방하다.</p> </blockquote> <ul> <li> <p>아래 프로젝트 파일을 다운받는다.</p> <p><a href="https://github.com/totuworld/firebase_toturial/archive/auth_guest.zip">샘플 프로젝트</a></p> </li> <li> <p><a href="https://developers.facebook.com/docs/unity">Facebook SDK for Unity</a>로 접속해서 Facebook Unity SDK를 다운로드한다(작성일 기준 v7.11.1).</p> </li> <li> <p>Unity 2017.3를 실행한 뒤 샘플 프로젝트를 열고 <a href="http://totuworld.github.io/2018/03/22/firebaseandunity-01/#프로젝트-설정">익명 로그인</a>의 <code>프로젝트 설정</code> 부분을 참고하여 <code>GoogleService-Info.plist</code> 파일을 생성하여 샘플 프로젝트에 추가한다. 다운로드한 Facebook Unity SDK도 추가한다.</p> <p>Facebook Unity SDK를 추가하면 Firebase Auth SDK와 충돌하는 파일이 4개 있다. 프로젝트에 사용된 Firebase Unity SDK는 4.3이고 Facebook Unity SDK는 7.11.1이다. 충돌이나는 파일은 <code>PlayServicesResolver의</code> 폴더에 있는 4개 파일인데 1.2.59 버전 파일을 모두 삭제한다.</p> <p><img src="/images/fb_03_02_01.png" alt="PlayServicesResolver 내 파일 삭제" width="80%" /></p> <blockquote> <p>Android 빌드는 확인해보지 않아서 어떤 문제가 발생할지 모릅니다.</p> </blockquote> </li> <li> <p>Unity에서 <code>Facebook</code> - href="http://totuworld.github.io/2016/12/21/azureandunity-01/">이세계에 진입한 서버 개발</a>을 시작하고 1년이 지났다. 그 동안 이직을 했고 바쁜 시간을 보냈다는 핑계로 업데이트 없는 9개월을 보냈다.</p> <p>놀기만 했으면 좋으련만 6월부터 <code>이세계에 진입한 서버 개발</code>을 책으로 출판할 생각으로 작업중이었는데 최근 절필했다. 2가지 이유가 있다. 하나는 오래된 코드라서 봐줄 수 없었다(이건 내 실력이…). 다른 하나는 게임 클라이언트 프로그래머만 있는 소규모 게임 개발팀에서 node.js와 관계형 데이터베이스를 공부해서 게임용 웹 서버를 만들 수 있을까하는 의문이다.</p> <blockquote> <p>게임 개발할 시간도 없는데 서버는 젠장!</p> </blockquote> <p>Wendy를 활용하는 게임 클라이언트 프로그래머가 인프라 공부에 시간 보낼까봐 PaaS(Platform as a Service)를 사용하도록 유도했지만 충분한 대답이 아니었다. 이보다 더 낮은 진입이 가능해야 순수 클라이언트 프로그래머가 공부할 수 있을 듯 했다. 그렇다면 좋은 서비스는 Parse나 Firebase같은 BaaS(Backend as a Service)다.</p> <blockquote> <p>Facebook이 인수하여 서비스하던 Parse는 서비스가 종료되었다.</p> </blockquote> <h3 id="firebase의-약팔이-새로운-기능">Firebase의 <del>약팔이</del> 새로운 기능</h3> <p>과거의 Firebase는 모바일 게임용 백엔드로 활용하기에 문제가 있었다. 게임 클라이언트에 Firebase SDK를 활용해 비지니스 로직을 넣게된다. 운영중에 비지니스 로직에 문제가 발생하면 모바일 게임은 플랫폼 심사 시간이 있어 빠른 대처가 어렵다. 출시 후 운영중에 대처가 늦으면 사용자 감소와 함께 매출이 감소한다. 이런 부담은 QA로 해소될 수 있으나 소규모 게임 제작팀은 비용때문에 불가능하다.</p> <p>2017년 3월 Firebase에 node.js 환경으로 프로그래밍이 가능한 <code>Cloud Functions</code>이 추가되었다. 이 기능을 활용하면 비지니스 로직을 게임 클라이언트 외부로 옮길 수 있다. 또 Firebase에 <code>Realtime Database</code>외에도 <code>Cloud Firestore</code>란 데이터베이스가 추가되었다. 이제 거대한 파일 하나에 모든 데이터를 넣지 않아도 된다. 단, 아직까지 Firebase Unity SDK에서는 사용할 수 없다.</p> <blockquote> <p>MS에서도 <a href="http://Playfab.com">Playfab</a>을 인수하여 Google의
