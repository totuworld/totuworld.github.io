들어가는 말

고백

이세계에 진입한 서버 개발을 시작하고 1년이 지났다. 그 동안 이직을 했고 바쁜 시간을 보냈다는 핑계로 업데이트 없는 9개월을 보냈다.

놀기만 했으면 좋으련만 6월부터 이세계에 진입한 서버 개발을 책으로 출판할 생각으로 작업중이었는데 최근 절필했다. 2가지 이유가 있다. 하나는 오래된 코드라서 봐줄 수 없었다(이건 내 실력이…). 다른 하나는 게임 클라이언트 프로그래머만 있는 소규모 게임 개발팀에서 node.js와 관계형 데이터베이스를 공부해서 게임용 웹 서버를 만들 수 있을까하는 의문이다.

게임 개발할 시간도 없는데 서버는 젠장!

Wendy를 활용하는 게임 클라이언트 프로그래머가 인프라 공부에 시간 보낼까봐 PaaS(Platform as a Service)를 사용하도록 유도했지만 충분한 대답이 아니었다. 이보다 더 낮은 진입이 가능해야 순수 클라이언트 프로그래머가 공부할 수 있을 듯 했다. 그렇다면 좋은 서비스는 Parse나 Firebase같은 BaaS(Backend as a Service)다.

Facebook이 인수하여 서비스하던 Parse는 서비스가 종료되었다.

Firebase의 약팔이 새로운 기능

과거의 Firebase는 모바일 게임용 백엔드로 활용하기에 문제가 있었다. 게임 클라이언트에 Firebase SDK를 활용해 비지니스 로직을 넣게된다. 운영중에 비지니스 로직에 문제가 발생하면 모바일 게임은 플랫폼 심사 시간이 있어 빠른 대처가 어렵다. 출시 후 운영중에 대처가 늦으면 사용자 감소와 함께 매출이 감소한다. 이런 부담은 QA로 해소될 수 있으나 소규모 게임 제작팀은 비용때문에 불가능하다.

2017년 3월 Firebase에 node.js 환경으로 프로그래밍이 가능한 Cloud Functions이 추가되었다. 이 기능을 활용하면 비지니스 로직을 게임 클라이언트 외부로 옮길 수 있다. 또 Firebase에 Realtime Database외에도 Cloud Firestore란 데이터베이스가 추가되었다. 이제 거대한 파일 하나에 모든 데이터를 넣지 않아도 된다. 단, 아직까지 Firebase Unity SDK에서는 사용할 수 없다.

Firebase 맛보기

앞서 소개한 기능을 다루기전에 Unity에서 Firebase를 얼마나 쉽게 사용할 수 있는지 체험해보자.

프로젝트 추가

Firebase 프로젝트 추가에 앞서 Firebase를 사용하려면 google 계정이 필요하다.

  • google 계정으로 로그인 한다.
  • Firebase console로 이동한다.

  • 프로젝트 추가를 클릭한다.

  • 프로젝트 이름을 입력하고 국가/지역을 선택한 뒤 프로젝트 만들기를 클릭한다.

프로젝트 설정

Firebase를 클라이언트(Android, iOS, Web)에서 사용하려면 설정을 추가하면 된다.

여기서는 iOS로 설명하지만 Android라고 크게 다를건 없다.

  • Project Overview 메뉴 옆에 기어 모양 설정 버튼을 클릭하고 프로젝트 설정을 클릭한다.

  • 일반 탭에서 하단의 iOS 앱에 Firebase 추가 클릭한다.

  • iOS 번들 ID를 입력하고 나머지 선택사항도 필요에 따라 입력한다.

  • GoogleService-Info.plist 파일을 다운받고 계속 버튼을 눌러 등록 과정을 마친다.

Firebase Unity SDK 설치

  • Firebase Unity SDK 페이지로 이동 후 SDK 다운로드 클릭

  • Unity 클라이언트 실행 후 Assets - Import Package - Custom Package를 클릭

  • FirebaseDatabase.unitypackage를 선택하여 추가.

필요에 따라 다른 package도 추가하면 된다.

  • Unity 프로젝트에 GoogleService-Info.plist 파일을 추가한다(어느 폴더여도 상관없다).

  • GoogleService-Info.plist을 추가하면 팝업 메뉴가 나온다. Apply 버튼을 클릭하여 Bundle ID가 적용되도록 한다.

데이터 추가

Firebase의 실시간 데이터베이스에 users란 노드를 생성하고 nick_name과 locale을 가진 데이터를 2개 추가하겠다.

  • Database를 선택한 뒤 Realtime Database 혹은 실시간 데이터베이스를 선택한다.

  • root node에 마우스 오버하면 +와 x 버튼이 나온다. + 버튼을 클릭해서 아래처럼 데이터를 입력한다.

Text 추가

Unity 클라이언트가 실행되면 users의 fox 데이터를 읽어서 nick_name을 화면에 출력할 것이다.

그러려면 Text를 하나 추가해야하니 이제 Unity로 돌아가서 test란 Scene을 생성한다.

프로젝트 생성하면 기본 제공하는 Scene을 그대로 사용해도 무방하다.

  • GameObject - UI - Text를 선택하여 Text를 추가한다.

  • 추가된 Text를 선택(Canvas 게임 오브젝트의 자식 게임 오브젝트로 생성됨)하고 Inspector에서 Pos X, Pos Y를 조절하여 화면 중앙에 위치시킨다.

간단한 프로그래밍

2개의 스크립트 파일을 추가할 것이다. 하나는 users의 데이터와 동일한 필드를 보유한 데이터 클래스. 나머지 하나는 실시간 데이터베이스의 데이터를 수신해서 Text에 출력하는 클래스이다.

  • Project에서 Assets를 선택한 뒤 마우스 오른쪽 버튼을 클릭하여 Create - C# Script 선택한 뒤 gfb_test로 입력한다.

  • 같은 방법으로 User도 추가한다.

추가된 User 스크립트를 더블 클릭하면 코드 에디터가 실행되어 프로그래밍할 수 있다.

강좌에서는 VS Code를 사용하도록 설정했다.

코드 에디터에 User 내용이 나타나면 아래처럼 변경한다.

public class User {
  public string nick_name;
  public string locale;

  public User() {
  }

  public User(string nickName, string locale) {
    this.nick_name = nickName;
    this.locale = locale;
  }
}

nick_name과 locale을 필드로 가지는 데이터 클래스이다.

이제 User 클래스를 활용해서 데이터가 변경되면 변경된 데이터의 nick_name을 출력하도록 해보자.

gfb_test 스크립트를 아래처럼 변경하자.

using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.UI;
using Firebase;
using Firebase.Database;
using Firebase.Unity.Editor;

public class gfb_test : MonoBehaviour
{
  public Text m_MyText;

  void Start()
  {
    // Set this before calling into the realtime database.
    FirebaseApp.DefaultInstance.SetEditorDatabaseUrl("https://YOUR-FIREBASE-APP.firebaseio.com/");

    // 사용자 로딩.
    DatabaseReference userReference = FirebaseDatabase.DefaultInstance.GetReference("users");

    // 하위 노드 변경 이벤트 처리하도록 등록.
    userReference.ChildChanged += HandleChildChanged;
  }

  /** snapshot 데이터에서 닉네임을 추출하여 text로 출력한다. */
  void PrintNickName(DataSnapshot snapshot) {
    User convertUser = JsonUtility.FromJson<User>(snapshot.GetRawJsonValue());
    m_MyText.text = string.Format("nick_name: {0}", convertUser.nick_name);
  }

  /** users의 하위 노드가 수정된 이벤트를 수신하여 처리 */
  void HandleChildChanged(object sender, ChildChangedEventArgs args) {
    if (args.DatabaseError != null) {
      Debug.LogError(args.DatabaseError.Message);
      return;
    }
    Debug.Log(string.Format("ChildChanged:{0} \n {1}", args.Snapshot, args.Snapshot.GetRawJsonValue()));
    PrintNickName(args.Snapshot);
  }
}

Start 메서드에서 users 노드를 참고하도록 설정한 뒤 하위 노드 변경이 발생하는 이벤트를 처리할 수 있도록 HandleChildChanged와 연결했다.

HandleChildChanged 메서드는 변경점이 발생하면 log 메시지를 출력하고 m_MyText에 nick_name을 출력한다.

YOUR-FIREBASE-APP 부분은 자신의 Firebase 프로젝트 url을 넣어야한다.

스크립트 적용

Unity 프로젝트로 돌아가서 Main Camera 게임 오브젝트에 gfb_test 스크립트를 추가하고 text를 연결해보자.

  • Main Camera 게임 오브젝트를 선택하고 Proejct 탭에서 gfb_test를 선택하여 Main Camera 게임 오브젝트에 끌어다 놓는다.

  • My Text에 Text를 끌어다 놓는다.

프로젝트 실행

프로그래밍된 결과를 확인해보자.

  • Unity에서 Play 버튼을 클릭한다.

  • 최초에는 New Text란 글자가 Game 탭에 출력된다.

  • Firebase 실시간 데이터베이스에서 tororo나 fox의 nick_name을 변경한다.

  • Unity의 Game 탭에 변경한 데이터가 출력된다.

정리
